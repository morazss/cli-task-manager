# CLI Task Manager

Консольный менеджер задач на Go. Позволяет добавлять задачи, просматривать список и отмечать их как выполненные прямо из терминала. Задачи хранятся локально в базе данных [BoltDB](https://github.com/boltdb/bolt).

## Структура проекта

```
task/
├── go.mod
├── main.go        # точка входа, инициализация БД и запуск CLI
├── cmd/
│   ├── root.go    # корневая команда task
│   ├── add.go     # команда add
│   ├── list.go    # команда list
│   └── do.go      # команда do
└── db/
    └── db.go      # BoltDB: Init, CreateTask, AllTasks, DeleteTask
```

## Установка

```bash
git clone <ссылка-на-репозиторий>
cd task
go mod tidy
go build -o task .
```

После сборки можно переместить бинарник в `$PATH`, чтобы вызывать `task` из любого места:

```bash
mv task /usr/local/bin/
```

## Использование

### Добавить задачу

```bash
task add Купить продукты
# Added "Купить продукты" to your task list.

task add Написать отчёт по проекту
# Added "Написать отчёт по проекту" to your task list.
```

### Просмотреть все задачи

```bash
task list
# You have 2 following tasks:
# 1: Купить продукты
# 2: Написать отчёт по проекту
```

### Отметить задачу(и) как выполненную

```bash
task do 1
# Marked "1" as completed.

# Несколько задач за раз:
task do 1 2 3
```

## Как это работает

Точка входа `main.go` при старте:
1. Определяет домашнюю директорию пользователя через `go-homedir`.
2. Инициализирует BoltDB по пути `~/task.db` (файл создаётся автоматически при первом запуске).
3. Запускает CLI через `cobra`.

Команды `add`, `list`, `do` реализованы в пакете `cmd` через `cobra.Command` и подключены к корневой команде `RootCmd` через `init()`.

Пакет `db` реализует четыре операции над BoltDB:

| Функция | Описание |
|---|---|
| `Init(dbPath)` | Открывает БД, создаёт бакет `tasks` если не существует |
| `CreateTask(task)` | Записывает задачу, ключ — автоинкремент (`NextSequence`) |
| `AllTasks()` | Возвращает все задачи в порядке добавления |
| `DeleteTask(key)` | Удаляет задачу по ключу |

`do` намеренно удаляет задачу по её **оригинальному ключу в БД**, а не по порядковому номеру в списке — это гарантирует, что нумерация в `list` и индекс для `do` всегда консистентны.

## Зависимости

| Пакет | Назначение |
|---|---|
| `github.com/spf13/cobra` | CLI-фреймворк, команды и флаги |
| `github.com/boltdb/bolt` | Встроенная key-value БД (без сервера) |
| `github.com/mitchellh/go-homedir` | Кроссплатформенный путь к домашней директории |

## Возможные улучшения

- Команда `rm` для удаления задачи без отметки «выполнено»
- Флаг `--priority` для приоритизации задач
- Команда `done` для просмотра завершённых задач
- Поддержка нескольких списков задач (`task list work`, `task list personal`)
